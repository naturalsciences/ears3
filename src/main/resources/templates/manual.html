What is EARS

EARS stands for “Eurofleets Automatic Reporting System” and is a tool allowing to log events such as sampling events, gear deployments or instrument manipulations and to couple the events with the underway data (i.e. navigation, thermosalinometry and meteorology parameters). The concept is as follows: the user defines the event e.g. start of sampling with Niskin bottles and submits the event while the sampling takes place. The event is logged with its exact date and time and coupled with the acquisition data of the same time (e.g. latitude and longitude, depth, surface water temperature, salinity, conductivity, solar radiation,…) which are continuously recorded by the vessel equipment, by finding the nearest value for each. In addition, events can be edited afterwards, including time, or missed events can be added and edited retroactively at the end of the cruise.

The software has been developed mainly by RBINS-BMDC in the framework of the Eurofleets projects 2 and +.

Terminology

General

  • Cruise or campaign: a scientific journey of a research vessel, with a start and end harbour, led by a chief scientist.
  • Program: a specific scientific research project performed during a cruise, led by a principal investigator. This can be a repeated activity across many campaigns (even for many years), or specific to a single cruise. In this case you can use the same name for both cruise and program.

Event

An event is some action that happens at specific time with a scientific device. The person executing it and the research program are also taken into account.

Events are composed of the following terms:

  • Tool category: groups of sensors, instruments or sampling devices (e.g. sediment profile imagers, beam trawls)
  • Tool: the specific equipment performing the event (e.g. sampling gear, instrument, sensor, observational device, research vessel). Tools can belong to one or more tool categories (e.g. Kongsberg (Simrad) TOPAS PS018 sub-bottom profiler is both a sub-bottom profilers and a chirper)
  • Process: a general activity a tool can do and consists of a sequence of punctuated actions (including malfunctions and deliberate actions) (e.g. Track)
  • Action: a step in an ongoing process, and can be deliberate (e.g. Start) or incidentally (e.g. Failure)

Access to EARS

The application is only accessible onboard the vessel from a local IP address.

The url at any RV is as follows: http://<IP-ADDRESS>:<PORT>/ears3 and is specific to each EF+ Research Vessel. Ask your local electronician for the correct location.

The EARS web application can be used with any internet browser. At the moment, the application does not work properly with the Chrome browser.

Note that the figures included in this document are taken from the instance running on the RV Belgica and the shown ip address will not be the same on another vessel.

Using EARS

General

On first use, the user’s first and last name as wel as a valid email address are requested, as well as their affiliation. The list of organizations is extensive and (for now) unsorted. Start typing the name of the organization. If the organization does not appear, try searching for the full/abbreviated name. These settings can ben changed at any moment by clicking on the provided first and last name in the navigation bar.

Event creation

Before the user can start submitting a new event, the Current program needs to be selected. If you try to create an event without program, an error is shown.

Drop-down lists show the available tool categories, tools, processes and actions. A combination of these selections describes an event. The contents of the drop-down lists are pre-defined in EARS. In case the equipment is not available in the selection list, it has to be requested beforehand to a data manager who can pre-define it before the cruise and make it available on the vessel EARS server.

To create a new event, define first the station name (ex: MOW1) as well as a label for the event (ex: st1 bodem). Both are optional but are strongly recommended. In a next step both are copied to the event’s text boxes, where they can be further edited if needed. This way you don’t have to repeatedly type them for each event.

To create the event, four terms need to be selected. They can be selected in any order.

  • Select the Category: e.g. discrete water sampler, sediment grabs
  • Select the Tool: e.g. Niskin bottle, Van Veen grab
  • Select the Process: e.g. Sampling
  • Select the Action: e.g. Start

Once you select a term, the other terms available become more limited. For instance, if you first select Sampling, only devices capable of sampling become available.

When the four terms are selected, click on the Submit button. A pop-up window will appear confirming the station name and the event label. A Description can be added as well from the pop-up window. Click again on Submit when done. When the submit button turns to green, the event is created and logged in the vessel database of EARS.

When a new event is created, a shortcut scenario button is automatically created in the Common scenarios panel with the selected terms. This scenario button can be used as shortcut to log events with the same terms (tool category, tool, process and action). The actor, time stamp, station, label and description may be different of course. To log an event from the Common scenarios, click the desired scenario, define the station, label and description if needed and click on the Submit button.

It is possible to discard a scenario by clicking x on the scenario button. This does not delete the events generated from it.

Accessing the logged events

The Events tab of the web application allows to access the list of logged events. The user can:

  • Select and display the events details
  • Edit the event
  • Delete an event

The logged times are in UTC.

To display the event details, click on the desired event. The event description will open, displaying the station, label, date, time and its features. You can immediately modify the event if needed.

The Current program needs first to be re-defined. Then, the event station, label, date and time fields can be updated directly. To update the terms (tool, tool category, process or action), first unlock the corresponding red lock. When ready, click on Submit.

Events can be edited or created at any time during the cruise. However, it is strongly recommended to log the events while the action is actually performed. Missed events can still be added or updated retroactively during the cruise but beware, the EARS application and database are only accessible on board of the vessel. This means that it is not possible to make any updates when you have left the vessel after the cruise.

Show current navigation information

This function is not directly related to the EARS web application, but can be useful.

For complementary information on the vessel navigation, it is possible to get access to the near real-time en-route data that gives information on:

  • Time
  • Current position of the ship
    • Longitude
    • Latitude
  • Heading
  • Speed over water (sow)
  • Depth
  • Course over ground (cog)
  • Speed over ground (sog).

On /ears3Nav/nav/getLast/xml, the latest navigational information is shown in xml format as illustrated below. You can also use the /ears3Nav/nav/getLast/json or /ears3Nav/nav/getLast/datagram endpoints to view the data as json and csv respectively.

Export

ALL events

To export logged events, go to Events and click on “Download all events”. All events created so far in EARS are exported in a csv file. After opening the csv file (with excel for instance), you may select events of your cruise by filtering by the cruise’s period.

Events linked to a cruise

When you select the Cruises tab, you can export all events from the selected cruise. Please verify after exporting the data, that the export was successful!

Points of attention and tips

Points of attention:

  • Time in EARS is in UTC. This is important to couple the events with navigation and acquisition data. Always enter the time in UTC.

  • Any creation or editing action is only successfully executed when the Submit button (or Common scenario button) turns green for a couple of seconds. When this does not happen, verify that all fields are filled in (e.g. program, station, label).

Tips:

  • If the needed tool/scenario is not existing yet in EARS, select a similar one and add a meaningful label or description with the correct tool/scenario. Do not forget to request the tool to BMDC for next time.
